Freelance Project

Employee Monitoring System Web Application Design
Case Study

Employee Monitoring System is an application which helps to monitor its employees and also allows the employees to request leaves and manage their profiles.

  • Project Duration : February 16 2022 - February 18 2022 (3 days)
  • My Role: Solo UX/UI Designer

Problem Statement
Create wireframes for an Employee Monitoring System (Web Application)

1. Dashboard should contain quick links and basic functionalities.
2. Admin should be able to monitor all the employees.
3. Admin should be able to assign leaves.
4. Users should be able to request for leave.
5. Users should be able to see/edit their profile.

The Solution
A web application which can cater to different users based on their login.

1. Admin UI - The admin can monitor, view, edit and delete details of its company's employees.
2. Employee UI - The employee can view, edit and delete his/her details and also raise leave requests, monitor job growth, and finances, etc.

UI Screens
Dashboard consisting of a general overview and quick links

• Admin can see and overview of the company's employees and also navigate to the details by clicking on the metrics
• Admin can monitor statistics such as, Employee status, Payroll distribution, App usage etc.

Leave request overview

• Admin can view the list of employees requesting for leaves and approve/reject them.
• He can also filter the Leave requests based on Department, Request date, From date, Till date, Number of days, Leave type etc.

Employee leave request

• An employee can select From and To date, Leave type and add notes to request leaves.
• An employee can also seen an overview of his/her team's approved leave request overview.

Employee's view profile

• An Employee can view all his/her details pertaining to his/her employment with the company in this screen.
• An employee can edit his/her details, view the Job role and descriptions, Company hierarchy, Documents pertaining to his job role etc.

Conclusion + Learnings
What I’d do differently next time.

This is my first time creating a Employee Monitoring User Interface. I learnt a lot of information while working on this assignment. I gathered a lot of information by looking at the different company applications and contacted a few colleagues to understand how such tools work in their organisation.
If I had more time,

Follow proper UX rituals -

TI would spend quality time in researching about my users, create personas, conduct researches and user interviews. This would have helped me to empathise my users a lot more and understand their needs better.
If I had more time,

Foresee better navigation structure -

I would be able to create a proper task-flow diagram to view and ideate the entire navigation structure of this application.
I would also create extensive clickable prototype.

Build for the end-users -

I would also be conduct usability tests to validate my solutions with the end-users so that I could clarify my assumptions thoroughly.
